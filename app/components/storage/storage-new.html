Block(RBD) storage is recommended for tranditional applications that expect natvie(block) storage. It integrates with Kernel Virtual Machines(KVMs) and allows mounting of block storage devices from Linux or QEMU KVM clients. Data access viar Librados, RBD, and CephFS (POSIX complaint).

In storage for OpenStack, data is stored as objects. This type of storage is recommended for OpenStack deployments and unstructured data. It is ideal for massive scale, when the number of objects are very large. Object metadata enables searching or mining of very large data sets. Objects can be organized using S3 buckets or Swift containers though data is stored in a flat structure. Data is primarily accessed at the application level via RESTful API, HTTP(or URL), S3, or OpenStack Swift data is also accessible via OpenStack storage services.
